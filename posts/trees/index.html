Trees

Posted on Oct 31, 2023 DRAFT

Concepts

A tree is a data structure which stores elements hierarchically; A tree is made out of a set of nodes connected by edges. Every tree has a single root node, which is a node without a parent node.

A parent node points to other nodes; Nodes pointed to are called children nodes.

Nodes with no children are called leaf nodes; Nodes with children are interior nodes.

Any node can be considered the root of a subtree.

There is a single path from the root to any node; Path length equals number of edges traversed.

Ascendants and Descendants

Ascendents of node 12: 50, 90, 75.

Descendants of node 90: 50, 37, 12.

Height and depth

Height of a node: maximum path length from node to a leaf.

Depth of a node: path length from the root to that node.

Degree

Degree of a node is equal the number of its children.

Binary Trees

A binary tree is a tree whose nodes have only 2 child references; left and right.

Full BT and Perfect BT

A full binary tree is a binary tree in which every node is a leaf or has exactly two children.

  • has n + 1 leaves, where n is internal nodes

A perfect binary tree is a full binary tree in which all leaves have the same depth.

  • has 2h + 1 - 1 nodes, where h is height

Kd-trees

Kd-tree can be a “2d-tree”, where k is the number of dimensions.

A Kd-tree is a binary tree that stores a finite set of points from a k-dimensional space.

Kd-trees are useful for searches involving a multidimensional search key, like nearest neighbour searches or range searches.
