Linux Suck-Less Audio

Posted on Aug 13, 2024

Understanding Current Audio Solutions

There are three commonly discussed audio abstraction layers in Linux:

  1. Pulseaudio
  • Created as a modern replacement for Enlightenment Sound Daemon (ESD), it provides advanced features like per-application volume control and network audio streaming.
  1. Jack
  • Designed as a professional audio server for Linux, it offers low-latency audi processing and precise control, making it suitable for professional audio work.
  1. Pipewire
  • The newest abstraction layer, Pipewire integrates seamlessly with both Pulseaudio and Jack. It also includes built-in Bluetooth support, eliminating the need for separate tools like bluez (bluetoothctl).

Understanding the Stack

  1. Kernel layer

The kernel layer provides hardware abstraction. Advanced Linux Sound Architecture (ALSA) is the primary component that interacts with sound cards and manages communication, replacing the older Open Sound System (OSS).

  1. Middleware/Backend layer

ALSA Library (libasound): Provides a userspace interface to interact with ALSA.

Problem: ALSA is specific to Linux. What about other operating systems?

Solution: The need for cross-platform support led to the creation of sound servers like Pulseaudio.

Although these systems were created out of necessity, end users might not need an entire abstraction layer if they can use the kernel’s sound system directly.

Key Point: For a more minimalistic setup, consider using ALSA directly to avoid the complexity of additional layers.

Problems Frequently Associated and Solutions

  1. Software Compatibility with Pulseaudio

Issue: Some software is designed specifically for Pulseaudio. How can you use it with ALSA? Solution: Use the apulse package, which emulates Pulseaudio on top of ALSA.

Software designed to work directly with ALSA will function without issues.

  1. Bluetooth Device Connectivity

Issue: ALSA does not include built-in Bluetooth support. Solution: Install the bluez package along with its utilities, and use bluez-alsa to handle Bluetooth audio routing through ALSA.

Proper configuration

Instead of dumping it all here, I will provide my ALSA configurations that are on my .dotfiles repo under alsa directory: https://github.com/jpnt/.dotfiles

Note on Pipewire: I no longer recommend using Pipewire due to its high coupling and dependencies, such as requiring dbus, a session manager, and XDG_RUNTIME_DIR (usually set by logind).
