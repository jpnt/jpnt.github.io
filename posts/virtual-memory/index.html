Virtual Memory

Posted on Jul 26, 2023 DRAFT

Concepts

Physical memory is a list of bytes where each byte has an address.

Virtual memory is an abstraction layer between the address space and physical memory. So when a program uses an address, that address does not correspond to the bytes in the actual physical location of memory; This means all addresses a program uses are virtual.

Page table is a data structure used by the OS to map virtual addresses to the corresponding physical addresses in memory.

x86_64 address spaces

The x86_64 architecture is 64-bit, registers (and addresses) are 64 bits wide.

But, virtual addresses on current x86_64 processors only have 48 meaningful bits.

That is why, when you print an address, for example in C, you only get 12 digits after “0x” and not 16 as you should if you had 64 bit address spaces.
